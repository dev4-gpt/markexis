## Part 9 — The Architecture Decision That Matters (90 seconds)

**Say — this demonstrates system thinking:**

> "The thing I'm most deliberate about is the LLM stack. I'm not using OpenAI or Anthropic. Every AI call runs on Groq's free tier — 14,400 requests per day, under 3 seconds per lead.
>
> The fallback chain is Groq → Cerebras → Sambanova → NVIDIA NIM → OpenRouter. If one rate-limits, the next picks up automatically. Total AI cost: zero dollars per month.
>
> For images, instead of Midjourney or DALL-E, I use Cloudflare's free Workers AI — Flux Schnell, 10,000 neurons per day free, returns a JPEG in 2 seconds. Same 4-deep fallback chain.
>
> For embeddings, Jina AI gives 1 million tokens per month free. The entire RAG stack — 768-dimensional vectors, ivfflat index, cosine similarity search — costs zero.
>
> A system like this only works at scale if the cost per operation is nearly zero."

## Likely Questions

**"How is this different from Apollo or Clay?"**
> "Apollo is $1,200/month. Clay is $800/month. I built equivalent ICP scoring for $6/month. More importantly, the Groq classifier is tuned to Markexis's specific ICP — it knows to flag LatAm signals, AI implementation signals, and funding stages. That's not configurable in Apollo."

**"You said all 5 pillars are live — what exactly works end-to-end?"**
> "All 18 workflows are deployed and active. Pillar 0: scrapers run, leads are being scored, outreach.js sends emails. Pillar 1: keyword research runs, SerpBear tracks positions. Pillar 2: single command produces a full article with social posts and image in 13 seconds. Pillar 3: daily competitor crawl is scheduled. Pillar 5: RAG knowledge base is live, pipeline orchestrator chains everything. What's not done: video production (MoneyPrinterTurbo needs a Pexels API key), LinkedIn DM routing, and Reddit/ProductHunt scrapers pending API approval."

**"What happens when LinkedIn blocks the scraper?"**
> "The scraper uses stealth Playwright, random 3–12 second delays, max 80 profiles per session, business hours only, and reuses session cookies so LinkedIn doesn't see repeated logins. If a session gets flagged, we delete the cookie and re-warm the account."

**"Why n8n and not Zapier or Make?"**
> "n8n runs on a $6 VPS with no per-execution limits. Make caps at 10,000 operations per month. But the real reason is the Code node — I can run arbitrary async JavaScript with full error handling. The entire Firecrawl + Groq + Supabase pipeline runs in a single node with precise control."

**"What's the NocoDB/Open WebUI setup?"**
> "Both self-hosted on the same $6 VPS. NocoDB connects directly to the Supabase Postgres via the connection pooler — live read/write access to all 6 tables. Open WebUI runs Llama 3.3 70B via Groq with 8 Python tools wired to n8n webhooks. Access both via SSH tunnel — one command opens both in the browser."

**"What are the next builds?"**
> "LinkedIn DM routing (Phase 6 in the plan), video production via MoneyPrinterTurbo, and the LinkedIn multi-agent system — engagement alerts, ICP finder from post reactions, LinkedIn coach. All designed, infrastructure is ready."
